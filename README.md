## Мета роботи
Ознайомитися з основними можливостями бібліотеки **NumPy** для створення, обробки та аналізу масивів.  
Порівняти ефективність реалізації завдань із використанням **універсальних функцій NumPy**  
та **традиційних циклів Python**.


- Кожне завдання реалізовано **у двох варіантах**:
  - через **універсальні функції NumPy**;
  - через **цикли for** у чистому Python.  
- Для обох варіантів вимірювався **час виконання** за допомогою модуля `time`.

## Результати
За результатами замірів часу:

- У **деяких завданнях** (`1`, `3`, `4`, `15`)  
  бібліотека **NumPy** виконувала операції помітно швидше.  
- Проте **в інших випадках** (наприклад, `6`,`12`, `10`, `17`, `18`)  
  різниця у швидкості була **мінімальною або відсутня**.  
- Таким чином, ефективність NumPy **залежить від типу обчислень**:  
  при роботі з великими матрицями він має перевагу,  
  але для малих масивів виграш не завжди очевидний.

  
##  Висновок
У результаті виконання лабораторної роботи:
- було засвоєно принципи створення та обробки масивів у Python;
- вивчено способи використання **NumPy** для математичних операцій;
- проведено порівняння з реалізацією через цикли.

**Перевага NumPy не завжди значна** — у деяких завданнях швидкість виконання обох підходів була **приблизно однаковою (50/50)**.  
Проте NumPy забезпечує більш **зручний синтаксис** і легше масштабується для обробки великих даних.
