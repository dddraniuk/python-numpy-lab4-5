## Результати

### Завдання 1, 3, 4, 6, 10, 12, 15, 17, 18
За результатами обчислень:
- У ряді завдань (1, 3, 4, 15) NumPy виконував операції **швидше**, завдяки внутрішній векторизації.
- У завданнях 6, 12, 10, 17, 18 різниця у швидкості була **мінімальною**, оскільки матриці/масиви мали малий розмір, і накладні витрати на запуск NumPy нівелювали перевагу.

### Завдання 3 (Обчислення матричного виразу)
Два способи:

- Цикли (початковий рівень)

## Висновок

У процесі виконання роботи було:
- засвоєно принципи створення, модифікації та аналізу масивів у Python;
- вивчено переваги векторизованих обчислень у NumPy;
- виконано порівняння продуктивності різних підходів.

Зроблено висновок, що:
- **NumPy найбільш ефективний при роботі з великими масивами та матрицями**, де його оптимізовані алгоритми демонструють значний виграш у часі.
- При **малих розмірах даних** різниця між NumPy та циклами **може бути незначною**, а інколи цикли навіть швидші.
- Тим не менш, NumPy забезпечує **компактність, читабельність та масштабованість коду**, що робить його кращим вибором для роботи з даними.
